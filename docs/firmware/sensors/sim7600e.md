SIMCom SIM7600E LTE Cat-1 modem providing cellular PPP uplink. Managed by `CellularManager`, not the sensor pipeline.

## Transport

- UART1 (default, configurable via `Kconfig` and `CellularConfig`)
- RX: GPIO18, TX: GPIO17 (default — shared with GPS NMEA default, cannot be used simultaneously)
- Baud rate: 115200
- DTE RX buffer: 4096 bytes, TX buffer: 512 bytes

| Signal | GPIO (default) | Configurable |
|--------|---------------|-------------|
| UART RX (ESP32 ← modem TX) | GPIO18 | Kconfig / CellularConfig |
| UART TX (ESP32 → modem TX) | GPIO17 | Kconfig / CellularConfig |
| PWRKEY | GPIO12 | Kconfig / CellularConfig |
| SLEEP / DTR | GPIO21 | Kconfig / CellularConfig |
| RESET | — | CellularConfig (0xFF = not wired) |

GPIO defaults are set in `Kconfig.projbuild` and written into `CellularConfig` on first boot. They can be overridden at runtime via NVS (Device configuration page).

## Initialization sequence

Performed by `CellularManager::attemptConnect()` on every connection attempt:

1. Allocate PPP netif (`esp_netif_new` with `ESP_NETIF_DEFAULT_PPP`)
2. Configure and create DTE (`esp_modem_create_uart_dte`)
3. Create SIM7600E DCE (`esp_modem_new_dev_sim7600`)
4. Set APN via `esp_modem_set_apn` (PDP context)
5. Unlock SIM PIN via `esp_modem_set_pin` if `sim_pin` is non-empty
6. Poll registration state (`AT+CEREG?` via esp-modem) and CSQ every 2 s; state `2` ("searching") keeps polling without failure escalation
7. Set PPP PAP auth via `esp_netif_ppp_set_auth` if `username` is non-empty
8. Enter PPP data mode via `esp_modem_set_mode(ESP_MODEM_MODE_DATA)`
9. Wait for IP event (up to 30 s)
10. Run ICMP connectivity check if `connectivity_check_host` is non-empty
11. Monitor PPP in bounded 25 s waits; on `kLostIpBit` tear down, or force disconnect after two consecutive failed liveness probes

## Reconnect behaviour

- On failure: table backoff of 10 s, 30 s, 1 min, 2 min, 5 min, 10 min, then 15 min cap
- After 2 min of continuous failure: enter the hard retry tier and log the transition
- After 10 min of continuous failure: PWRKEY hardware reset is allowed only if the last PWRKEY cycle was at least 1 h ago
- If a continuous failure window would need more than two PWRKEY cycles, the firmware escalates to `esp_restart()`
- PWRKEY power-off pulse: 3500 ms; power-on pulse: 2000 ms; boot wait: 5000 ms
- After PPP is connected, silent link death is detected with a 25 s liveness cadence against `connectivity_check_host`; two consecutive failed one-ping probes force a disconnect and reconnect cycle.

## Runtime state

Exposed via `CellularManager::state()` and reflected in the web UI:

| Field | Meaning |
|-------|---------|
| `enabled` | Cellular uplink is configured |
| `modem_detected` | DTE/DCE created successfully |
| `sim_ready` | SIM not locked or PIN accepted |
| `registered` | Network registration complete |
| `ppp_connected` | Active PPP session with assigned IP |
| `ip_address` | Assigned PPP IP address |
| `rssi_dbm` | Signal strength in dBm (from AT+CSQ, converted) |
| `connectivity_ok` | Last ICMP ping result |
| `connectivity_check_skipped` | True when `connectivity_check_host` is empty |
| `consecutive_failures` | Current continuous cellular setup failure count |
| `pwrkey_cycles_total` | Total successful PWRKEY cycles since boot |
| `last_pwrkey_uptime_ms` | Uptime timestamp for the last successful PWRKEY cycle |
| `last_error` | Reason string for the last failure |

## Built-in GNSS

The SIM7600E contains an integrated GNSS receiver. It is **not currently used** by the firmware.

Possible integration paths:

- **Dedicated GNSS UART** — the modem exposes a separate NMEA output pin (`GNSS_TX`). If wired to a spare ESP32 UART (e.g. UART2), the existing `gps_nmea_sensor.cpp` driver could read it without modification.
- **AT command query** — `AT+CGNSINF` / `AT+CGPSINFO` return position data, but AT commands are unavailable while in PPP data mode. This path is only viable before PPP is established.

The dedicated GNSS UART approach is preferred. It operates independently of the cellular PPP session and is compatible with the existing sensor pipeline.

## Notes

- **GPIO17/18 conflict:** The default UART pins overlap with the GPS (NMEA) sensor default. GPS and cellular cannot be used simultaneously on UART1. Move the GPS sensor to UART2 in the Sensor Configuration page, or reassign the modem UART before enabling both.
- The modem task stack is 8192 bytes (priority 5). PPP netif, DTE, and DCE together consume significant heap — avoid additional large heap allocations in the cellular task.
- The DTE internal task (`esp_modem_uart_dte`) runs at default priority with a 4096-byte stack.
