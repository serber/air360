# DS18B20

Digital temperature sensor from Dallas/Maxim using the 1-Wire protocol. Supports up to 12-bit resolution (0.0625 °C step).

## Transport

- GPIO (1-Wire bus)
- Pin configured from the sensor record (`record.analog_gpio_pin`)
- Valid pins: GPIO4, GPIO5, GPIO6
- Pull-up enabled in the 1-Wire bus configuration
- Uses the **RMT peripheral** of the ESP32-S3 for precise 1-Wire timing

## Initialization

1. Verify that `analog_gpio_pin >= 0`
2. Create a 1-Wire bus via `onewire_new_bus_rmt()`:
   - GPIO pin from the sensor record
   - Pull-up enabled
   - RMT config with `max_rx_bytes = 10`
3. Create a device iterator via `onewire_new_device_iter()`
4. Enumerate all devices on the bus:
   - For each device, attempt `ds18b20_new_device_from_enumeration()`
   - If a second DS18B20 is found: return `ESP_ERR_NOT_SUPPORTED`
5. Delete the iterator
6. Verify exactly one DS18B20 was found
7. Set resolution to 12-bit via `ds18b20_set_resolution(DS18B20_RESOLUTION_12B)`

## Polling

Each poll cycle:
1. Trigger a temperature conversion via `ds18b20_trigger_temperature_conversion()`
2. Read temperature via `ds18b20_get_temperature()` — blocks until conversion completes

## Measurements

| Measurement | ValueKind | Unit |
|-------------|-----------|------|
| Temperature | `kTemperatureC` | °C |

## Notes

- **Resolution is hardcoded at 12 bits** — conversion time is ~750 ms. This waiting time is included in every poll and is the dominant cost of each sensor read.
- **Only one device per GPIO pin is supported.** 1-Wire theoretically allows multiple devices on a single bus, but the driver explicitly rejects this and returns `ESP_ERR_NOT_SUPPORTED` if more than one DS18B20 is found during initialization.
- The RMT peripheral handles 1-Wire timing without blocking the CPU, unlike software-based implementations.
- The device ROM address is stored at initialization but is not re-validated on each poll.
- If either `ds18b20_trigger_temperature_conversion()` or `ds18b20_get_temperature()` returns an error, `initialized_` is reset.

## Recommended poll interval

Minimum 5 seconds. Each poll takes at least 750 ms due to the 12-bit conversion time.
