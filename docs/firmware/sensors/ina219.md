DC current, voltage, and power monitor from Texas Instruments with an I2C interface. Measures via a shunt resistor placed in series with the load.

## Transport

- I2C, bus 0 (SDA=GPIO8, SCL=GPIO9)
- Default address: `0x40`
- Allowed addresses: `0x40`, `0x41`, `0x44`, `0x45` (set by A0/A1 pads on module)
- Clock: 100 kHz

## Hardware

The INA219 requires a shunt resistor between VIN+ and VIN− to measure current. Standard breakout modules (such as the SAMIROB module) include a `100 mΩ` (R100) shunt resistor on the board — no external resistor is needed.

Wiring the module in series with the load:

```
Supply (+) → VIN+  [INA219 module]  VIN− → Load (+)
Supply (−) ──────────────────────────────── Load (−)
```

The INA219 GND pin must share ground with the ESP32 GND and the supply negative terminal.

## Initialization

1. Resolve bus pins via `context.i2c_bus_manager->resolvePins()`
2. Initialize the sensor descriptor via `ina219_init_desc()` (passes resolved port, GPIO pins, and I2C address from `record.i2c_address`)
3. Initialize the device via `ina219_init()`
4. Configure the sensor:
   - Bus voltage range: `INA219_BUS_RANGE_32V` (supports up to 26 V)
   - Gain: `INA219_GAIN_0_125` (±320 mV shunt range — covers up to ~3.2 A with 100 mΩ shunt)
   - Bus ADC resolution: `INA219_RES_12BIT_1S`
   - Shunt ADC resolution: `INA219_RES_12BIT_1S`
   - Mode: `INA219_MODE_CONT_SHUNT_BUS` (continuous measurement)
5. Calibrate via `ina219_calibrate()` with `r_shunt = 0.1 Ω` (100 mΩ)

## Polling

Each poll cycle:
1. Read bus voltage via `ina219_get_bus_voltage()` — returns volts; stored as mV
2. Read current via `ina219_get_current()` — returns amps; stored as mA
3. Read power via `ina219_get_power()` — returns watts; stored as mW

## Measurements

| Measurement  | ValueKind      | Unit |
|--------------|----------------|------|
| Bus voltage  | `kVoltageMv`   | mV   |
| Current      | `kCurrentMa`   | mA   |
| Power        | `kPowerMw`     | mW   |

## Notes

- The shunt resistance is hardcoded to `0.1 Ω` (100 mΩ), matching the standard R100 shunt on common INA219 breakout modules. If the module uses a different shunt value, this constant must be changed in the driver.
- Negative current readings indicate that the VIN+ and VIN− connections are swapped. Swap the wires at the green terminal to correct polarity.
- Gain `INA219_GAIN_0_125` allows up to ±320 mV across the shunt, which corresponds to approximately ±3.2 A with a 100 mΩ shunt. Higher currents will saturate the ADC.
- The INA219 bus voltage measurement is referenced to the VIN− pin relative to GND. If GND is not shared with the supply negative, voltage readings will be incorrect.
- Motors and other inductive loads generate back-EMF spikes that can corrupt I2C transactions. If the firmware crashes under motor load, add a 100 nF ceramic capacitor across the motor terminals and ensure solid decoupling on the INA219 VCC line.
- If any read call returns an error, `initialized_` is reset and the sensor re-initializes on the next poll cycle.

## Recommended poll interval

Minimum 5 seconds. The INA219 in continuous mode updates its registers every few milliseconds; the 5 s floor is the Air360 system minimum.

## I2C address selection

| A1 pad | A0 pad | Address |
|--------|--------|---------|
| Open   | Open   | `0x40` (default) |
| Open   | Bridged to VCC | `0x41` |
| Bridged to VCC | Open | `0x44` |
| Bridged to VCC | Bridged to VCC | `0x45` |
