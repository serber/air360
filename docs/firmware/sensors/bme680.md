# BME680

Temperature, humidity, pressure, and gas resistance sensor from Bosch. The gas channel serves as an indirect air quality indicator (VOC).

## Transport

- I2C, bus 0 (SDA=GPIO8, SCL=GPIO9)
- Default address: `0x77`
- Valid addresses: `0x76`, `0x77`
- Clock: 100 kHz, pull-ups enabled

## Initialization

1. Initialize the `i2cdev` subsystem
2. Initialize the sensor descriptor via `bme680_init_desc()`
3. Initialize the sensor via `bme680_init_sensor()`
4. Configure measurement mode:
   - Oversampling: x2 for temperature, humidity, and pressure
   - IIR filter: `IIR_SIZE_0` (off)
   - Ambient temperature for gas calibration: 25 °C (hardcoded)
5. Configure the gas heater: profile 0, target temperature 300 °C, heating duration 100 ms
6. Activate heater profile 0

## Polling

Each poll cycle:
1. Calculate measurement duration via `bme680_get_measurement_duration()`
2. Trigger a forced measurement via `bme680_force_measurement()`
3. Wait `measurement_duration_ticks` via `vTaskDelay()`
4. Read results via `bme680_get_results_float()`
5. Validate that no NaN values are returned

## Measurements

| Measurement | ValueKind | Unit |
|-------------|-----------|------|
| Temperature | `kTemperatureC` | °C |
| Humidity | `kHumidityPercent` | % |
| Pressure | `kPressureHpa` | hPa |
| Gas resistance | `kGasResistanceOhms` | Ω |

Gas resistance is only included in the measurement if the returned value is > 0.

## Notes

- Measurement duration is calculated dynamically by the sensor library. Each poll blocks the sensor task for approximately 150–200 ms due to the heater warm-up (100 ms) plus conversion time.
- The ambient temperature used for gas calibration is hardcoded at 25 °C. This affects gas resistance accuracy in environments that differ significantly from room temperature.
- The heater profile (300 °C / 100 ms) activates the chemical reaction on the gas sensing element. Between polls the heater cools down.
- Uses `i2cdev` with GPIO pins passed directly into the descriptor, unlike drivers that work through `I2cBusManager`.
- If a read fails or returns NaN, `initialized_` is reset and the driver re-initializes on the next cycle.

## Recommended poll interval

Minimum 5 seconds. Each poll takes approximately 150–200 ms of blocking time inside the sensor task.
