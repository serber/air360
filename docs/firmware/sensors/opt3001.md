Ambient light sensor from Texas Instruments with an I2C interface. Air360 reports the current illuminance in lux.

## Transport

- I2C, bus 0 (SDA=`GPIO8`, SCL=`GPIO9`)
- Default address: `0x44`
- Allowed addresses: `0x44`, `0x45`, `0x46`, `0x47`
- Clock: 100 kHz, pull-ups enabled

## Hardware

Connect the module to the shared Air360 I2C bus:

```text
OPT3001 module  ->  ESP32-S3 / Air360
VCC                 3.3 V
GND                 GND
SDA                 GPIO8
SCL                 GPIO9
ADDR                board-specific address strap
INT                 not connected by Air360
```

- The bare OPT3001 supply range is `1.6..3.6 V`; use `3.3 V` on ESP32-S3 boards.
- Breakout-board pull-ups should be to `3.3 V`.
- `ADDR` selects the I2C address. Air360 defaults to `0x44` and allows the other OPT3001 address straps from the web UI.
- `INT` is not used; Air360 polls the sensor.

## Initialization

1. Set up an `i2c_dev_t` through `I2cBusManager::setupDevice()`.
2. Check that a device responds at the selected I2C address.
3. Read and validate manufacturer ID `0x5449` and device ID `0x3001`.
4. Mark the driver initialized.

## Polling

Each poll cycle:

1. Write the configuration register for automatic range, 800 ms conversion time, and single-shot mode.
2. Wait 825 ms for conversion completion.
3. Read the configuration register and verify that conversion is ready and the device returned to shutdown mode.
4. Read the result register.
5. Decode exponent/mantissa to lux: `lux = 0.01 * 2^exponent * mantissa`.

The driver records no value if conversion is not ready.

## Measurements

| Measurement | ValueKind | Unit |
|-------------|-----------|------|
| Illuminance | `kIlluminanceLux` | lux |

## Notes

- Air360 registers this as `SensorType::kOpt3001 = 20`.
- The driver uses direct I2C register access; no external managed component is required.
- The sensor is in the `Light` category with VEML7700, so only one light sensor can be configured at a time.
- Sensor.Community does not support this value in the Air360 adapter, so OPT3001 is skipped there. Air360 API, Custom Upload, and InfluxDB receive the generic `illuminance_lux` value.

## Recommended poll interval

30 seconds. The actual conversion delay inside each poll is about 825 ms.

## Component

None. The driver uses Air360 `I2cBusManager` and `i2cdev`.
