# BME280

Temperature, humidity, and pressure sensor from Bosch.

## Transport

- I2C, bus 0 (SDA=GPIO8, SCL=GPIO9)
- Default address: `0x76`
- Valid addresses: `0x76`, `0x77` (configurable through the web UI)

## Initialization

1. Verify I2C bus availability via `I2cBusManager`
2. Probe the I2C address to confirm device presence
3. Read the chip ID register and verify it matches `BME280_DEFAULT_CHIPID`
4. Create a component bus handle via `I2cBusManager::getComponentBus()`
5. Create the BME280 component driver handle via `bme280_create()`
6. Apply default settings via `bme280_default_init()`
7. Configure measurement mode:
   - Oversampling: x1 for temperature, humidity, and pressure
   - IIR filter: off (`BME280_FILTER_OFF`)
   - Standby duration: 0.5 ms
   - Mode: `FORCED` (triggered on each read)

## Polling

Each poll cycle:
1. Trigger a forced measurement via `bme280_take_forced_measurement()`
2. Read temperature, humidity, and pressure sequentially

## Measurements

| Measurement | ValueKind | Unit |
|-------------|-----------|------|
| Temperature | `kTemperatureC` | °C |
| Humidity | `kHumidityPercent` | % |
| Pressure | `kPressureHpa` | hPa |

## Notes

- `FORCED` mode means the sensor sleeps between polls and takes a single measurement on demand. This minimizes self-heating at the cost of a brief acquisition delay per poll.
- Oversampling x1 reduces acquisition time at the expense of slightly higher noise.
- If `bme280_take_forced_measurement()` or any read call returns an error, `initialized_` is reset and the driver will re-initialize on the next sensor manager cycle.

## Recommended poll interval

Minimum 10 seconds (enforced by the registry). Typical: 10–60 s.
