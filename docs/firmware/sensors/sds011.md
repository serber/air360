Laser particulate matter sensor from Nova Fitness. The driver uses the UART protocol and reports PM2.5 and PM10 mass concentration.

## Transport

- UART at `9600` baud, 8 data bits, no parity, 1 stop bit
- Default binding: UART2, RX=`GPIO16`, TX=`GPIO15`
- UART1 is selectable through the descriptor: RX=`GPIO18`, TX=`GPIO17`
- The driver uses `UartPortManager`; no managed component is required

## Hardware

Connect the module to a 5 V supply and cross UART TX/RX:

```text
SDS011  ->  ESP32
5V          5 V
GND         GND
TXD         RX GPIO (default GPIO16)
RXD         TX GPIO (default GPIO15)
```

- The sensor requires a stable `4.7..5.3 V` supply.
- UART logic is TTL-level `3.3 V`.
- The PWM outputs are not used by Air360.
- Keep the fan inlet and outlet unobstructed; airflow restriction affects readings.

## Initialization

1. Store the selected `SensorRecord`.
2. Open the selected UART through `context.uart_port_manager->open()`.
3. Request the default 4096-byte RX ring and a short UART event queue.
4. Send a wake command (`0x06`, work mode).
5. Set work period to `0` (continuous operation) so modules left in duty-cycle mode by other firmware keep responding.
6. Set reporting mode to passive/query mode.
7. Flush command replies and stale RX input.
8. Mark the driver initialized.

The driver does not create any FreeRTOS task. It runs inside the existing sensor manager polling task.

## Polling

Each poll cycle:

1. Drain UART events and reset the parser if an overrun was reported.
2. Send a wake command. This is harmless for an already-running module and is required for modules left in sleep mode.
3. Send the standard SDS011 query command.
4. Read available UART bytes with a 50 ms first-read timeout.
5. Parse 10-byte measurement frames:
   - header `0xAA`
   - command `0xC0`
   - PM2.5 low/high bytes
   - PM10 low/high bytes
   - two sensor ID bytes
   - checksum over the six data bytes
   - tail `0xAB`
6. Store the latest valid frame from the poll cycle.

The driver also accepts autonomous 1 Hz active-reporting frames already buffered by the UART. If no bytes arrive, or bytes arrive without a valid frame, the poll returns `ESP_OK` with `last_error_` set and keeps the previous measurement.

## Measurements

| Measurement | ValueKind | Unit |
|-------------|-----------|------|
| PM2.5 mass concentration | `kPm2_5UgM3` | ug/m3 |
| PM10 mass concentration | `kPm10_0UgM3` | ug/m3 |

Raw SDS011 values are reported in tenths of `ug/m3`; the driver divides the raw integer by `10.0`.

## Notes

- The restored Air360 support uses the legacy `SensorType::kSds011 = 7` value for compatibility with older SDS011 records.
- The firmware wakes the module, forces continuous work period, and uses passive/query reporting mode. This restores modules previously configured by firmware that puts SDS011 into sleep or duty-cycle mode between samples.
- UART read errors use the common soft-fail policy. After three consecutive poll errors, the driver marks itself uninitialized so `SensorManager` can re-initialize it with backoff.
- The SDS011 fan and laser are wear items. Continuous operation is documented around an 8000-hour service life, so duty-cycle management may be a future enhancement.

## Recommended poll interval

10 seconds. The sensor can emit data every second, but Air360 defaults to 10 seconds to keep upload volume and UI churn lower.

## Component

None. The driver uses ESP-IDF UART through Air360 `UartPortManager`.
